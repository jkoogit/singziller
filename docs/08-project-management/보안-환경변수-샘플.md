# 보안 환경변수 샘플

## 목적

실제 비밀번호와 서비스 보안 설정값은 Git에 커밋하지 않고, 서버 또는 로컬 환경의 `.env` 파일에서만 관리한다. 이 문서는 `.env`에 넣을 값의 형식과 확인 항목을 정리한다.

## Git 관리 원칙

- `.env`, `.env.*` 파일은 커밋하지 않는다.
- `.env.example`과 문서 샘플에는 실제 비밀번호를 넣지 않는다.
- 로컬 또는 Ubuntu 서버에서 직접 생성한 `.env`는 untracked 상태로 둔다.
- PR, 이슈, 작업 리뷰 문서에는 비밀번호 원문을 적지 않는다.

## 애플리케이션 서버 `.env` 샘플

저장 위치 예시:

```text
server/.env.development
server/.env.staging
server/.env.production
```

개발 환경 예시:

```env
APP_ENV=development
HOST=0.0.0.0
PORT=3000
DATABASE_URL=postgresql://devdbszusr:replace-with-password@postgres-dev:5432/singziller_dev
CORS_ORIGINS=http://localhost:4173
SESSION_SECRET=replace-with-random-secret-at-least-32-characters
JWT_SECRET=replace-with-random-secret-at-least-32-characters
```

검증 환경 예시:

```env
APP_ENV=staging
HOST=0.0.0.0
PORT=3000
DATABASE_URL=postgresql://stgdbszusr:replace-with-password@postgres-stg:5432/singziller_stg
CORS_ORIGINS=https://stg.example.com
SESSION_SECRET=replace-with-random-secret-at-least-32-characters
JWT_SECRET=replace-with-random-secret-at-least-32-characters
```

운영 환경 예시:

```env
APP_ENV=production
HOST=0.0.0.0
PORT=3000
DATABASE_URL=postgresql://prddbszusr:replace-with-password@postgres-prd:5432/singziller_prd
CORS_ORIGINS=https://example.com
SESSION_SECRET=replace-with-random-secret-at-least-32-characters
JWT_SECRET=replace-with-random-secret-at-least-32-characters
```

## PostgreSQL 컨테이너 `.env` 샘플

Ubuntu 서버 저장 위치:

```text
/opt/singziller/postgres/dev/.env
/opt/singziller/postgres/stg/.env
/opt/singziller/postgres/prd/.env
```

개발 환경:

```env
POSTGRES_USER=devdbszusr
POSTGRES_PASSWORD=replace-with-password
POSTGRES_DB=singziller_dev
POSTGRES_PORT=35432
POSTGRES_VOLUME=postSzDev
```

검증 환경:

```env
POSTGRES_USER=stgdbszusr
POSTGRES_PASSWORD=replace-with-password
POSTGRES_DB=singziller_stg
POSTGRES_PORT=45432
POSTGRES_VOLUME=postSzStg
```

운영 환경:

```env
POSTGRES_USER=prddbszusr
POSTGRES_PASSWORD=replace-with-password
POSTGRES_DB=singziller_prd
POSTGRES_PORT=55432
POSTGRES_VOLUME=postSzPrd
```

## 작성 전 체크리스트

- 비밀번호와 secret은 환경별로 다르게 만든다.
- `POSTGRES_PASSWORD`, `SESSION_SECRET`, `JWT_SECRET`은 32자 이상 랜덤 문자열을 사용한다.
- `DATABASE_URL`에 특수문자가 들어간 비밀번호를 넣을 때는 URL encoding이 필요하다.
- Docker 내부 네트워크에서는 DB host를 `postgres-dev`, `postgres-stg`, `postgres-prd` 같은 서비스명으로 둔다.
- 외부에서 직접 점검할 때만 `192.168.219.125:35432` 같은 host port를 사용한다.
- 운영 `CORS_ORIGINS`에는 실제 서비스 도메인만 넣는다.
- `.env` 작성 후 `git status --short`에 `.env` 파일이 보이지 않는지 확인한다.

## 안전 확인 명령

비밀번호를 출력하지 않고 키만 확인한다.

```bash
grep -E '^(POSTGRES_USER|POSTGRES_DB|POSTGRES_PORT|POSTGRES_VOLUME)=' /opt/singziller/postgres/dev/.env
```

Git에 실수로 올라갈 파일이 있는지 확인한다.

```bash
git status --short
git check-ignore server/.env.development
git check-ignore server/.env.production
```
