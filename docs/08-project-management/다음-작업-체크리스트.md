## 의사결정 필요

### 기술 스택

- 서버 런타임: Node.js/TypeScript를 우선 적용한다.
- API 프레임워크: Fastify를 우선 적용한다.
- migration: node-pg-migrate를 우선 적용한다.
- 배포 위치: Ubuntu 서버의 Docker 내부 네트워크를 기준으로 한다.
- DB 환경: `dev`, `stg`, `main` 브랜치에 맞춰 개발/검증/운영 PostgreSQL을 분리한다. 자세한 내용은 [DB 환경 구성](DB-환경-구성.md)을 따른다.

### 데이터 배포 포맷

- JSON: 구현이 쉽지만 큰 데이터에서 비효율 가능.
- NDJSON: streaming과 delta 처리에 유리.
- SQLite 파일: 오프라인 검색 성능이 좋지만 PWA 적용 난이도 상승.

초기 추천은 `JSON snapshot + JSON delta`이고, 데이터가 커지면 `NDJSON`으로 전환한다.

### 수집 정책

- 공식 사이트 크롤링을 할 것인지, 수동/개인 API 우선으로 시작할 것인지.
- TJ/KY 요청 빈도 기본값.
- YouTube API quota 사용 범위.
- 개인 API 제공자별 trust level 산정 기준.

### 운영/검수

- 수동 확정 데이터의 우선순위.
