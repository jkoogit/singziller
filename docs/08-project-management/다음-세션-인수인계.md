5. collection run 시작/성공/실패 기록 interface 작성.
6. source record 중복 저장 방지 흐름 작성.
7. 실제 DB 없는 repository mock 테스트 작성.
8. 이후 dev DB 통합 테스트 또는 migration 보강 여부 판단.

## 설계 기준

- 공식 TJ/KY 사이트 호출은 아직 구현하지 않는다.
- `tj-official`, `ky-official`, `youtube`, `personal-api`, `google-sheet`는 stub 또는 mock provider로 둔다.
- 외부 수집은 robots.txt와 이용약관 확인 후 진행한다.
- raw record 원본과 정규화 결과를 분리한다.
- 중복 판정은 provider별 raw hash를 기준으로 한다.

## 주의사항

- `server/.env` 값은 출력하지 않는다.
- DB 비밀번호와 secret은 PR, Issue, 리뷰 문서에 적지 않는다.
- `DATABASE_URL`의 비밀번호에 특수문자가 있으면 URL encoding이 필요하다.
- Windows 로컬에서 Docker 내부 hostname인 `postgres-dev`는 해석되지 않을 수 있다.
- 로컬에서 직접 검증할 때는 host를 `192.168.219.125:35432`로 바꿔 확인한다.
