# 요구사항 정의서

## 기능 요구사항

### 곡 검색

- 사용자는 제목, 가수, 노래번호, 브랜드, 분류, 버전, 링크 유무로 곡을 검색할 수 있다.
- 브랜드를 TJ로 선택하면 TJ 번호를 우선 표시한다.
- 브랜드를 KY로 선택하면 KY 번호를 우선 표시한다.
- 브랜드를 선택하지 않으면 TJ/KY 구분과 매핑 후보를 함께 표시한다.
- 같은 곡에 여러 KY 버전이 있으면 버전별로 표시한다.

### 곡정보 카탈로그

- 서비스 기준 곡정보는 PostgreSQL의 독자 카탈로그를 우선한다.
- PWA는 카탈로그 manifest를 확인하고 필요한 scope만 IndexedDB에 다운로드한다.
- 다운로드된 곡정보는 오프라인 검색에 사용한다.
- 카탈로그는 version과 checksum으로 무결성을 관리한다.
- delta 적용 실패 시 해당 scope snapshot을 다시 다운로드한다.

### 곡정보 수집

- 수집 모듈은 앱과 분리된 서버 작업으로 운영한다.
- 수집 대상은 TJ/KY 공식 페이지, YouTube, 개인 수집 API, Google Sheet, 수동 입력이다.
- 수집 원본은 raw record로 저장한다.
- 정규화/매칭/신뢰도 계산 후 song catalog 후보로 만든다.
- 신뢰도 낮은 후보는 검수 큐로 보낸다.

### 사용자 데이터

- 레파토리, 북마크, 녹음은 로컬 우선으로 관리한다.
- Google 계정 로그인을 지원하되 로그인 없이도 기본 기능을 사용할 수 있다.
- Google Sheets import/export를 지원한다.
- 사용자 수동 보정은 자동 수집 결과가 덮어쓰지 않는다.

### 녹음과 공유

- 사용자는 브라우저 마이크 권한으로 녹음할 수 있다.
- 녹음 파일은 로컬에 저장한다.
- 곡정보는 Web Share API 또는 클립보드로 공유할 수 있다.

## 비기능 요구사항

- PWA 구조를 유지한다.
- 오프라인에서도 다운로드된 곡정보와 사용자 데이터는 사용할 수 있어야 한다.
- 카탈로그 다운로드는 scope 단위로 선택 가능해야 한다.
- 검색은 모바일 환경에서도 빠르게 동작해야 한다.
- 외부 API 장애 시 로컬 데이터 사용성을 유지해야 한다.
- 테스트 주도 개발을 적용하고 작업별 리뷰 문서를 남긴다.

## 데이터 요구사항

### 곡정보 필수 필드

- 브랜드
- 노래제목
- 가수
- 노래번호
- 버전
- 분류
- 등록일시
- 수정일시
- 출처
- 신뢰도

### 선택 필드

- 작사가
- 작곡가
- YouTube 링크
- Spotify 링크
- 원본 URL
- 외부 ID
- 검수 상태

## 권한 요구사항

- Google Sheets 연동 시 Google OAuth Client ID가 필요하다.
- YouTube API 사용 시 API key 또는 OAuth 정책이 필요하다.
- 녹음 기능 사용 시 브라우저 마이크 권한이 필요하다.

## 의사결정 필요

- 사용자 계정 서버를 둘지, Google Sheets 연동만으로 시작할지 결정해야 한다.
- 카탈로그 API 인증 정책을 결정해야 한다.
- 녹음파일을 장기적으로 로컬만 둘지 클라우드 동기화할지 결정해야 한다.
- 검수 큐의 운영 주체와 권한을 결정해야 한다.

## 보완 필요

- 상세 검색 랭킹 정책.
- 카탈로그 scope 분류 기준.
- 사용자 데이터 백업/복원 정책.
- 수집 실패 알림 정책.
