# 운영 키워드 반영 리뷰

## 작업 목적

일반 작업 마무리와 브랜치 승급을 명확히 구분하기 위해 `#세션마무리`, `#브랜치승급` 운영 키워드를 브랜치 운영 프로세스에 반영한다.

## 관련 이슈

- #2 세션마무리와 브랜치승급 운영 키워드 반영

## 변경 파일

- `docs/08-project-management/브랜치-운영-프로세스.md`
- `docs/reviews/2026-06-10-operation-keywords-review.md`

## 의사결정

- 일반 작업은 `#세션마무리` 흐름으로 진행한다.
- 일반 작업 흐름은 `이슈 먼저 등록 → 작업 브랜치 → PR은 dev로만`을 따른다.
- `dev > stg`, `stg > main` 승급은 `#브랜치승급` 흐름으로 진행한다.
- 승급 작업에는 별도 PR을 만들지 않고 댓글로만 이력을 남긴다.
- 승급 댓글에는 포함 PR 번호를 반드시 표시한다.

## 테스트 / 검증

- 문서 변경이라 기능 테스트 영향은 없다.
- 기존 Sheets 테스트와 `app.js` 문법 검사를 실행한다.

## 남은 위험

- 승급 이력을 어느 이슈에 누적할지는 아직 확정되지 않았다.
- 승급 방식이 merge commit인지 fast-forward인지 저장소 운영 중 확정이 필요하다.

## 다음 작업

- 이후 세션 종료 요청은 `#세션마무리` 기준으로 이슈, 작업 브랜치, `dev` 대상 PR까지 진행한다.
- 승급 요청은 `#브랜치승급` 기준으로 PR 없이 댓글 이력만 남긴다.
