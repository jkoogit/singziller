# 브랜치 운영 프로세스 정리 리뷰

## 작업 목적

GitHub 원격 저장소 연결과 `main`, `stg`, `dev` 브랜치 운영 정책을 문서화하고, 앞으로 세션 작업을 이슈와 PR 중심으로 관리할 수 있게 한다.

## 관련 이슈

## 의사결정

- Codex 로컬 작업은 `dev` 브랜치를 기준으로 진행한다.
- 기능 작업은 이슈를 먼저 만들고, 작업 PR은 `dev`로 보낸다.
- `dev > stg`, `stg > main` 승급에는 별도 PR을 만들지 않는다.
- 승급 이력은 한글 댓글로 남기고 포함 PR 번호를 표시한다.
- 이슈, PR, 댓글, 리뷰 문서는 한글로 작성한다.

## 테스트 / 검증

- 문서 링크 확인.
- 기존 Sheets 테스트 실행.
- `app.js` 문법 검사 실행.
- Git 원격 브랜치 생성과 push 결과 확인.

## 남은 위험

- 승급 댓글을 어느 이슈에 누적할지 아직 확정되지 않았다.
- 실제 PR 머지 방식은 GitHub 저장소 설정에 따라 추가 확인이 필요하다.
