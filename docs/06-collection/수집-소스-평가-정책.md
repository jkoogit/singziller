# 수집 소스 평가 정책

## 목적

공식 노래 API가 없는 상황에서 Singziller가 사용할 수 있는 곡정보 원천을 평가하고, 수집한 데이터를 서비스 카탈로그에 반영하기 전 검증 기준을 정한다.

## 원칙

- 공식 홈페이지 검색 결과를 최우선 근거로 둔다.
- 개인 API와 오픈소스 데이터는 후보 생성과 교차 검증에만 사용한다.
- lyrics, 음원, 영상, 팬 콘텐츠 원문은 수집하지 않는다.
- 운영자가 확정한 수동 보정은 자동 수집으로 덮어쓰지 않는다.
- robots.txt, 약관, 요청 빈도, 차단 신호를 provider별로 기록한다.

## 확인된 소스

### TJ Media

- robots.txt: `/error/`, `/upload/`, `/resources/admin/` 제외. 일반 공개 페이지 접근은 허용 형태.
- 반주곡 검색: `https://www.tjmedia.com/song/accompaniment`
- 최신곡: `https://www.tjmedia.com/song/recent_song`
- 최신곡 페이지 내부 호출 후보: `/legacy/api/newSongOfMonth`
- 구형 `/tjsong/song_search*.asp` 경로는 서비스 점검 페이지로 응답하므로 사용하지 않는다.

### KY Entertainment

- robots.txt: `/_program/`, `/adm/`, `/lib/` 제외.
- 공식 홈페이지에서 노래방 플랫폼 페이지와 반주곡 신청 게시판이 확인된다.
- 일반 곡번호 검색 엔드포인트는 아직 확정되지 않았다. 확정 전까지 KY 자동 수집 결과는 service catalog에 반영하지 않는다.

## 신뢰 등급

1. Tier 1: 공식 홈페이지 검색 결과
2. Tier 2: 공식 최신곡, 차트, 신청/공지 페이지
3. Tier 3: 개인 API, 오픈소스 DB, 공개 정적 JSON
4. Tier 4: Google Sheet, 운영자 수동 입력

Tier 3 데이터는 공식 소스와 일치하거나 운영자 검수를 통과하기 전까지 서비스 확정 데이터로 쓰지 않는다.

## 샘플 수집 정책

- 최초 2주는 샘플 수집 기간으로 둔다.
- TJ는 고정 검색어 20개 이하로 parser와 변경 감지를 확인한다.
- KY는 공식 검색 URL이 확정될 때까지 구조 조사와 후보 페이지 샘플만 저장한다.
- 개인 API는 license, 최신성, schema, 공식 소스 일치율을 평가한다.
- 샘플 데이터는 `candidate_only` 상태로 저장하고 자동 서비스 반영을 금지한다.

## 수집 주기 결정

초기 기본 정책은 혼합 수집이다.

- 최신곡/차트: 주 1회
- 전체 검색 재검증: 월 1회
- 개인 API 후보 재평가: 월 1회 또는 upstream 변경 감지 시

2주 샘플 수집 후 다음 기준으로 조정한다.

- 주간 수집: 변경량이 전체 샘플의 1% 이상이거나 신곡/차트 변경이 주 1회 이상 감지될 때.
- 월간 수집: 변경량이 작고 parser 실패가 없을 때.
- pause: 403, 429, 약관 변경, 구조 변경으로 신뢰할 수 없는 결과가 나오면 즉시 중단.

## 검증 프로세스

1. raw record 저장
2. provider별 hash와 external id로 중복 제거
3. 제목, 가수, 곡번호, 버전, 국가/분류 정규화
4. 공식/개인/수동 소스 간 교차 검증
5. 신뢰도 점수 계산
6. 낮은 점수 또는 충돌 record는 검수 큐로 이동
7. 검증 통과 record만 candidate catalog로 승격
8. 운영자 확정 또는 충분한 공식 근거가 있을 때 master catalog 반영

## 역할별 의사결정

- Data Steward: source trust tier와 충돌 우선순위 승인.
- Collector Engineer: provider adapter, parser, 요청 제한 구현.
- Validation Lead: 정규화, 점수화, 검수 큐 기준 관리.
- Operations Lead: 수집 주기, 실패 알림, pause/resume 관리.
- Compliance Reviewer: robots.txt, 약관, 요청량, 재배포 위험 검토.

## 중단 조건

- provider가 403 또는 429를 반환한다.
- robots.txt 또는 약관이 수집 대상 경로를 금지한다.
- parser mismatch가 발생해 필수 필드 정확성을 보장할 수 없다.
- 개인 API의 license/source attribution이 불명확하다.
- 공식 또는 권리자가 삭제/중단 요청을 한다.
