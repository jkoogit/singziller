# Singziller Catalog Policy

## 목적

Singziller의 서비스 기준 곡정보는 PostgreSQL에 저장된 독자 곡정보 카탈로그로 둔다. TJ, KY, YouTube, 개인 수집 API, Google Sheet, 수동 입력은 모두 원천 소스이며, 앱은 이 원천을 직접 신뢰하지 않고 정규화/검수/버전관리된 카탈로그를 우선 참조한다.

## 기준 원칙

- 앱 본체는 곡정보 수집 로직을 포함하지 않는다.
- 수집기는 별도 모듈로 운영하고 PostgreSQL에 원천 데이터와 정규화 데이터를 저장한다.
- PWA는 필요한 범위의 카탈로그 스냅샷 또는 delta만 내려받아 IndexedDB에 저장한다.
- 사용자 데이터인 레파토리, 북마크, 녹음, 수동 보정은 원천 카탈로그와 분리한다.
- 공식 TJ/KY 데이터가 있으면 곡번호와 브랜드별 등록 정보의 최우선 근거로 둔다.
- YouTube는 곡번호 매핑 근거가 아니라 링크/버전/공식 영상 후보 보강용으로 사용한다.

## 데이터 흐름

```text
External Sources
  - TJ/KY official pages
  - YouTube Data API
  - Personal collection APIs
  - Google Sheet
  - Manual admin input
        |
        v
Collector Modules
        |
        v
PostgreSQL raw source records
        |
        v
Normalize / Match / Score
        |
        v
Song Catalog
        |
        v
Catalog Snapshot / Delta API
        |
        v
PWA IndexedDB local catalog
```

## 신뢰도 우선순위

1. 공식 TJ/KY 데이터
2. 운영자가 수동 확정한 데이터
3. 검증된 개인 수집 API
4. Google Sheet 검수 데이터
5. YouTube 링크/영상 후보
6. 사용자 임시 입력

신뢰도가 낮은 후보는 자동 반영하지 않고 검수 큐로 이동한다.

## 버전 정책

카탈로그는 배포 가능한 버전 단위로 관리한다.

```text
catalog_version: 2026.06.10.001
scope: full | popular | kpop | trot | 7080 | pop | children | tj | ky
format: json | ndjson | sqlite
checksum: sha256
created_at
```

PWA는 `manifest`를 확인한 뒤 필요한 scope만 다운로드한다.

## 다운로드 정책

- 최초 실행: 인기추천과 경량 검색 인덱스만 다운로드한다.
- 사용자가 선택 다운로드할 수 있는 scope: 전체, TJ, KY, K-POP, 트로트, 7080, 팝송, 동요, 인기추천.
- 온라인 검색은 서버 카탈로그를 우선하고, 로컬 캐시 결과를 즉시 표시한다.
- 오프라인 검색은 IndexedDB에 다운로드된 scope 안에서만 동작한다.
- delta 적용에 실패하면 해당 scope만 full snapshot을 다시 다운로드한다.

## 충돌 정책

- 공식 데이터와 개인 API가 충돌하면 공식 데이터를 우선한다.
- 운영자가 확정한 수동 보정은 자동 수집 결과가 덮어쓰지 않는다.
- 같은 곡의 TJ/KY 매핑은 `confirmed`, `candidate`, `rejected` 상태로 관리한다.
- 삭제/폐곡/번호 변경은 물리 삭제하지 않고 `active`, `deprecated`, `removed`, `unknown` 상태로 관리한다.

## 의사결정 필요

- 카탈로그 배포 포맷을 JSON/NDJSON/SQLite 중 무엇으로 시작할지 결정해야 한다.
- 공식 사이트 크롤링 허용 범위와 요청 주기를 정책으로 확정해야 한다.
- 개인 수집 API 제공자의 신뢰도 등급과 책임 범위를 정해야 한다.
- 운영자 검수 도구를 별도 관리자 화면으로 만들지, 초기에는 SQL/시트 기반으로 운영할지 결정해야 한다.
- 무료/개인 서비스로 시작할지, 서버 비용이 드는 API 서비스로 바로 갈지 결정해야 한다.

## 보완 필요

- TJ/KY 공식 사이트의 실제 요청 방식, robots.txt, 약관 확인.
- 금영/KY 공식 검색 접근 가능한 최신 URL 확인.
- YouTube API quota와 검색 조건별 비용 산정.
- 카탈로그 scope별 예상 데이터 크기 측정.
- 오프라인 다운로드 용량 제한과 자동 정리 정책 수립.
