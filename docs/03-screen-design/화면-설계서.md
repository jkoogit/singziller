# 화면 설계서

## 화면 목록

| 화면 | 목적 | 주요 기능 |
| --- | --- | --- |
| 검색 | 곡정보 검색 | 통합 검색, 상세 조건, 브랜드별 결과 |
| 라이브러리 | 추천/분류 곡 탐색 | 인기추천, K-POP, 트로트, 7080, 팝송, 동요 |
| 레파토리 | 자주 부르는 곡 관리 | 추가, 삭제, 순서 관리 |
| 북마크 | 모임/시트별 곡 묶음 | 그룹 생성, 곡 추가 |
| 녹음 | 노래 녹음 관리 | 녹음, 재생, 다운로드 |
| 동기화 | Google Sheets 연동 | 로그인, import/export, 업로드 |
| 곡정보 다운로드 | 로컬 카탈로그 관리 | scope 선택, 버전 확인, delta 적용 |
| 검수 큐 | 운영자 검수 | 후보 확인, 확정, 거절, 수동 매핑 |

## 검색 화면

### 구성

- 상단 검색 조건 패널.
- 브랜드 선택: 전체, TJ, KY.
- 검색 필드: 통합 검색, 제목, 가수, 노래번호.
- 필터: 분류, 버전, 링크 유무.
- 정렬: 관련도, 제목, 가수, 최근 수정, 등록 버전 많은 순.
- 결과 카드: 브랜드, 번호, 제목, 가수, 버전, 매핑 후보, 링크, 레파토리/북마크 버튼.

### 상태

- 검색 결과 있음.
- 검색 결과 없음.
- 온라인 검색 중.
- 로컬 카탈로그만 사용 중.
- 해당 scope 미다운로드.

## 곡정보 다운로드 화면

### 구성

- 현재 설치된 카탈로그 버전.
- 서버 최신 버전.
- 다운로드된 scope 목록.
- 선택 가능한 scope 목록.
- 다운로드/업데이트 버튼.
- 저장 용량 표시.

### 주요 흐름

1. 앱이 `/catalog/manifest`를 조회한다.
2. 사용자에게 최신 버전과 보유 scope를 보여준다.
3. 사용자가 필요한 scope를 선택한다.
4. delta 또는 snapshot을 다운로드한다.
5. checksum 검증 후 IndexedDB에 반영한다.

## 검수 큐 화면

### 구성

- 수집 후보 목록.
- 원천 소스 정보.
- 정규화 결과.
- 기존 곡 매칭 후보.
- 신뢰도와 매칭 이유.
- 확정, 거절, 보류, 수동 병합 버튼.

### MVP 포함 여부

검수 큐는 서버/관리자 기능이므로 PWA MVP에는 포함하지 않는다. 초기에는 DB 상태값과 SQL/시트 기반 운영으로 시작할 수 있다.

## 모바일 설계 원칙

- 검색 조건은 접히는 패널로 전환 가능해야 한다.
- 결과 카드는 번호와 브랜드가 가장 먼저 보이게 한다.
- 레파토리/북마크 액션은 한 손 조작이 가능해야 한다.
- 다운로드 화면은 저장 용량과 오프라인 가능 범위를 명확히 보여준다.

## 의사결정 필요

- 곡정보 다운로드 화면을 기존 동기화 탭에 넣을지 별도 탭으로 만들지 결정해야 한다.
- 검수 큐를 PWA 내부 운영자 모드로 만들지 별도 관리자 앱으로 만들지 결정해야 한다.
- 검색 결과에서 TJ/KY를 카드 하나에 묶을지 브랜드별 카드로 분리할지 결정해야 한다.

## 보완 필요

- 실제 화면 와이어프레임.
- 모바일/데스크톱 breakpoint 정의.
- 접근성 기준.
- 오류/빈 상태 문구 확정.
