# 서비스 설계서

## 전체 아키텍처

```text
PWA Client
  - Search UI
  - IndexedDB local catalog
  - User data storage
  - Google Sheets import/export

Catalog API Server
  - Manifest API
  - Snapshot API
  - Delta API
  - Search API

PostgreSQL
  - Raw source records
  - Normalized candidates
  - Song catalog
  - Catalog versions

Collector Workers
  - TJ/KY official source collectors
  - YouTube collector
  - Personal API collector
  - Google Sheet collector
```

## 클라이언트 설계

PWA는 온라인/오프라인을 모두 지원한다.

- 온라인: 서버 검색 결과와 로컬 캐시 결과를 병합한다.
- 오프라인: IndexedDB에 저장된 scope만 검색한다.
- 사용자 데이터: 레파토리, 북마크, 녹음은 로컬 우선으로 관리한다.
- Google Sheets: 사용자 import/export 보조 기능으로 사용한다.

## 서버 설계

서버는 PWA가 직접 PostgreSQL을 알 필요 없도록 API를 제공한다.

- `/catalog/manifest`: 최신 버전과 scope별 다운로드 정보를 제공한다.
- `/catalog/snapshots/latest`: scope별 전체 snapshot을 제공한다.
- `/catalog/delta`: 버전 간 변경분을 제공한다.
- `/search`: 온라인 검색을 제공한다.
- `/sources/status`: 수집 상태를 제공한다.

## 수집 설계

수집기는 source별 adapter로 분리한다.

```text
CollectorProvider
  - 수집 실행
  - raw payload 저장
  - 중복 hash 계산
  - 수집 실행 이력 기록
```

정규화와 매칭은 수집기에서 바로 확정하지 않고 pipeline에서 처리한다.

## 데이터 갱신 흐름

1. 수집기가 source record를 저장한다.
2. normalizer가 제목/가수/버전/번호를 정규화한다.
3. matcher가 기존 song master와 후보 매칭한다.
4. scorer가 신뢰도를 계산한다.
5. 높은 신뢰도는 자동 반영하고 낮은 신뢰도는 검수 큐로 보낸다.
6. publisher가 catalog version과 delta를 생성한다.
7. PWA가 manifest를 확인하고 필요한 변경분을 다운로드한다.

## 장애 대응

- 수집 실패: collection run에 기록하고 다음 주기에 재시도한다.
- API 장애: PWA는 로컬 IndexedDB 검색으로 fallback한다.
- delta 적용 실패: snapshot 재다운로드로 복구한다.
- checksum 불일치: 적용을 중단하고 기존 버전을 유지한다.

## 보안과 권한

- read-only catalog API는 공개 가능하지만 rate limit을 둔다.
- 운영자 검수와 수집 실행은 인증이 필요하다.
- Google OAuth token은 PWA localStorage에 저장하지 않는다.
- YouTube API key는 서버에서 관리하고 클라이언트에 노출하지 않는다.

## 의사결정 필요

- API 서버와 수집기를 같은 배포 단위로 둘지 분리할지 결정해야 한다.
- catalog snapshot을 API 서버에서 직접 제공할지 object storage/CDN으로 제공할지 결정해야 한다.
- 검색엔진을 PostgreSQL로 시작할지 별도 검색엔진을 도입할지 결정해야 한다.

## 보완 필요

- 서버 배포 환경.
- 인증/권한 상세 정책.
- 운영 모니터링 지표.
- 백업/복구 정책.
